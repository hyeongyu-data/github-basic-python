# Peer Review 실행 가이드

코드/diff 리뷰에 사용하는 가이드입니다. 계획 문서 리뷰는
`agent-plan-review.md`를 사용합니다.

## 이 문서를 볼 때

- 변경이 여러 모듈이나 경계에 걸칠 때
- 동작·설정·워크플로가 바뀌었을 때
- 커밋이나 PR 생성 전 최종 품질 점검이 필요할 때
- PR 리뷰를 수행할 때

## 기본 리뷰 관점

| 관점 | 초점 |
| --- | --- |
| Critic | 기존 패턴에 맞는가, 의도치 않은 동작 변경이 없는가 |
| Code quality | 구성이 규칙을 따르는가, 중복·불필요한 복잡도가 없는가 |
| Convention | 이름, 주석, 문서, 커밋 형식이 저장소 규칙을 따르는가 |
| Security | 시크릿, 자격 증명, 입력 검증, 워크플로 권한이 안전한가 |

보안 경계(인증·시크릿·권한·외부 입력)가 바뀌었으면 Security 관점을 반드시
포함합니다.

## 리뷰 초점

심각도 순으로:

1. 정확성 버그와 기존 동작의 의도치 않은 변경
2. 시크릿·자격 증명 노출 위험
3. 입력 검증 누락, 권한 확대
4. 불필요한 복잡도·중복 (기존 코드로 대체 가능한지)
5. 테스트 누락

## 리뷰 프롬프트 템플릿

```text
현재 diff를 critic, code quality, convention, security 관점에서
리뷰하라. 의도치 않은 동작 변경, 시크릿 노출, 입력 검증 누락, 불필요한
복잡도, CLAUDE.md 및 .claude/docs/*.md 위반에 집중하라. 발견 사항을
파일·라인 참조와 함께 심각도 순으로 반환하라. 발견 사항이 없으면 남은
검증 리스크를 명시하라.
```

## 출력 규칙

- 발견 사항을 심각도 순으로 먼저 제시합니다.
- 파일과 라인 참조를 포함합니다.
- 칭찬은 나열하지 않습니다. 요약보다 구체적 발견이 우선입니다.
- 구체적 이슈는 인라인 코멘트로, 요약 코멘트는 짧게 유지합니다.
