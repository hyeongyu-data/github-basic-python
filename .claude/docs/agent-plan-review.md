# Plan Review 실행 가이드

구현 전 계획(plan) 문서를 리뷰할 때 사용하는 가이드입니다. 코드나 diff
리뷰는 `agent-peer-review.md`를 사용합니다.

## 이 문서를 볼 때

- 신규 기능 또는 대규모 다중 파일 변경
- 데이터 모델, 인증, 외부 연동 등 되돌리기 어려운 변경
- "계획 리뷰"나 모호성 점검이 명시적으로 요청된 경우

## 리뷰 질문

계획을 다음 질문으로 평가합니다:

1. 목표가 명확하고 범위가 한정되어 있는가?
2. 비목표(non-goal)가 범위 확장을 막을 만큼 명시적인가?
3. 영향받는 파일·모듈과 경계가 정확한가?
4. 구조 변경과 동작 변경이 분리되어 있는가?
5. 검증 방법이 구체적인가? (실제 실행 가능한 명령·테스트인가)
6. 되돌리기(롤백)와 데이터 마이그레이션 리스크가 다뤄져 있는가?
7. 가정(선행 작업, 준비 조건)이 명시되어 있고 확인 가능한가?
8. 요청된 결과에 비해 과잉 설계된 단계는 없는가? (YAGNI)

## 출력 형식

다음 순서로 반환합니다:

1. Blocking 이슈
2. Non-blocking 개선 사항
3. 모호한 부분과 가정
4. 제안하는 계획 수정

계획이 타당하면 그렇다고 말하고, 남아 있는 가장 큰 리스크를 나열합니다.
